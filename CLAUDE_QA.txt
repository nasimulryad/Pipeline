# QA Review Instructions

You are a senior React developer reviewing the Pipeline job search app.

Please do the following:

1. Run `git diff HEAD~1 --name-only` to see what files changed in the last commit
2. Run `git log -1 --oneline` to get the commit info
3. Check the `qa-reports/` folder — if a report already exists for this commit, stop and say "QA already completed for this commit"
4. For each changed JS/JSX file, review it and its direct imports
5. Check for: bugs, performance issues, security concerns, best practice violations, improvement opportunities
6. Create a new file in `qa-reports/` named `report-COMMITHHASH-YYYY-MM-DD-HH-MM.md`
7. Write a detailed report to that file with these sections:
   - Commit Reviewed
   - Files Reviewed
   - Summary
   - Bugs Found (with line numbers)
   - Performance Issues
   - Security Concerns
   - Improvement Recommendations
8. When the report is written and saved, type /exit to quit.

Be specific and actionable. Reference exact line numbers. Focus on meaningful issues only.
